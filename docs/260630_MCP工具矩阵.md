## 1. 约束规则

| 规则 | 说明 |
|------|------|
| **一系统一 MCP Server** | mes / scada / erp / lims / qms / wms / eam / knowledge / plc |
| **Agent 只 bootstrap 白名单** | `AgentCard.allowed_tools` ⊆ 对应 `mcp_servers` 下的 Tool |
| **Tool 命名** | `{server}.{tool_name}`，如 `mes.query_batch_trace` |
| **敏感 Tool** | `erp.query_recipe`、`plc.*` 需 RBAC；`plc` 写操作 **仅 safety-agent** |
| **Router 不调 MCP** | 只写 PlatformContext；MCP 由各业务 Agent 进程内 Harness 调用 |

**状态标记**：`implemented` 姊妹仓已有 · `in_process` RCA 进程内（未走 MCP）· `planned` 契约预留

## 5. bootstrap 实现要点

```python
# 伪代码：各 Agent 启动时
card = registry.get_card("trace-agent")
for server in card.mcp_servers:
    client = await MCPClient.connect(server)
    for tool in await client.list_tools():
        full = f"{server}.{tool['name']}"
        if full in card.allowed_tools:
            registry.register(ToolSpec(name=full, ...))
```

姊妹仓 RCA 当前仅连接 mes/scada/erp/lims 四域；`knowledge` 仍走进程内 FMEARegistry，与 `allowed_tools` 中 `knowledge.*` 对齐为 P1 任务。

## 6. 给同事的交付

认领某 Agent 时，只需对齐三列：

1. `AgentCard.mcp_servers`
2. `AgentCard.allowed_tools`（本文件 §3）
3. 若新增 Tool：先加到 **MCP Server**，再更新本矩阵，**禁止** Agent 私有直连 L3
