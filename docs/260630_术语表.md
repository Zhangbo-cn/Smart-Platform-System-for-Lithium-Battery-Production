# 术语表

避免把网关、编排器、Worker 误称为 Agent。

## Agent（唯一标准）

**Agent**：运行时存在 **LLM + Tool Use 循环**——模型在多轮推理中**自主决定**调用哪些 Tool、是否继续、何时结束。

不满足上述条件，**不得**在文档、职责描述、架构图中称为 Agent。

| 类型 | 定义 | 决策主体 | 示例 |
|------|------|----------|------|
| **Tool** | MCP 工具、函数；输入→输出 | 无 | `qms.create_8d_draft` |
| **Agent** | LLM + Prompt + Tool Use（± Memory） | 节点内 LLM | `quality-rca-agent`（LangGraph）、`planner-agent`（P1） |
| **Generator** | 单次 LLM 生成，无 Tool 循环 | LLM 一次 | RCA 内 Reporter 润色 |
| **Worker** | 确定性逻辑，可按固定顺序调 MCP | 代码 | `report-reporter-agent` 服务、trace 壳 |
| **Rule engine** | 规则 / 正则 / 查表 | 代码 | `triage_stub.resolve_triage()` |
| **Workflow** | 节点按拓扑编排（DAG / 状态机 / 有环图） | 开发者 | RCA 内 LangGraph |
| **Orchestrator** | 按 Playbook 逐步委派 A2A 服务 | Playbook 代码 | `router` |
| **Gateway** | 统一入口、转发、SSE、HITL 回写 | 无 | `client-agent` 服务 |
| **Registry** | 能力服务登记与探活 | 无 | `registry` |

## 平台控制面（均非 Agent，除非注明）

| 组件 | 服务目录 | LLM | 正确称呼 |
|------|----------|:---:|----------|
| 用户门户 | `client-agent/` | ❌ | **Client Gateway** |
| 剧本编排 | `router/` | ❌ | **Playbook Orchestrator**（文档可简称 Router） |
| 服务目录 | `registry/` | ❌ | **Capability Registry** |
| 任务规划 | `planner-agent/` | ✅（P1） | **Planner**（实现后才是 Agent） |

控制面组合：`Client Gateway` → `Planner`（可选）→ `Playbook Orchestrator` → 业务能力服务。

## 业务能力服务

| 服务 | LLM + Tool 循环 | 正确称呼 |
|------|:---------------:|----------|
| `quality-rca-agent` | ✅ | **Agent**（对外一个 A2A 服务，对内 Workflow） |
| `planner-agent` | ✅（P1 待实现） | **Agent** |
| `report-reporter-agent` | ❌（模板 + 固定 MCP） | **Worker** |
| `trace-agent` | ❌（计划为 MCP 壳） | **Worker** |
| `triage-agent` | ✅ LLM+Rule 双模 | **Agent**（独立进程 8001） |
| patrol / process / safety 等（仅 Card） | — | **能力占位**，非 Agent |

## A2A 协议用语（与 Agent 定义分离）

| 术语 | 含义 |
|------|------|
| **AgentCard** | A2A 协议中的**服务元数据**（name、url、skills）；登记名可含 `-agent` 后缀，**不等于**上文定义的 Agent |
| **A2A Protocol** | 服务间 JSON-RPC 风格 Task 通信 |
| **AgentNetwork** | name → url 的服务发现表（SmartVoyage 同名概念） |
| **MCP** | Tool 层协议；Worker / Agent 均可调用 |

> 实现约定：`AgentCard.name`（如 `client-agent`）为**稳定服务 ID**，保留历史后缀；文档与架构图使用 Gateway / Orchestrator / Worker 等准确称呼。

## 关系口诀

- **Agent**：仅指 LLM + Tool Use 循环的实现（如 RCA、Planner）。
- **A2A 服务**：独立部署、有 URL 的能力单元；含 Agent、Worker、Gateway、Orchestrator。
- **平台**：Gateway + Orchestrator + Registry + Session + Harness/MCP；业务能力服务由 Orchestrator 星型委派，**服务间不网状互连**。

## 三层状态

| 层级 | 作用域 | 谁消费 |
|------|--------|--------|
| **A2A TaskState** | 单次 A2A Task | Gateway、Orchestrator、驾驶舱 |
| **LangGraph State** | RCA **内部** | RCA Workflow 节点 |
| **PlatformContext** | 一次 session 跨服务 | Orchestrator、下游 Worker（如 8D） |

## 不宜混用的说法

| 避免 | 改用 |
|------|------|
| Client Agent | **Client Gateway** |
| AgentRouter（暗示是 Agent） | **Playbook Orchestrator** / Router |
| 每个 Workflow 节点都是 Agent | Agent 节点 / Worker 节点 / 规则节点 |
| N 个 Agent 微服务 | N 个 **A2A 能力服务**（其中仅部分为 Agent） |
| Multi-Agent = 自由聊天 | 任务式协作 + Playbook 编排 |
| report-reporter-agent 是 Agent | **8D Worker**（服务 ID 可仍为 `report-reporter-agent`） |

## 实现选型（对外部署形态）

| 模块特征 | 内部实现 | 对外 A2A 服务 | 文档称呼 |
|----------|----------|---------------|----------|
| 有环补查 + HITL | LangGraph | `quality-rca-agent` | Agent |
| 开放任务分解 | ReAct + Tool | `planner-agent` | Agent |
| 短链路 / 规则 / 模板 | MCP + 确定性代码 | `trace-agent`、`report-reporter-agent` | Worker |
| 停线 / 改参门闩 | 白盒规则 | `safety-agent` | Worker（规则门闩） |
