# A2A 协议规范 — 锂电生产智能化平台

> 本平台 A2A（Agent-to-Agent）借鉴 Google A2A 协议思想，采用 **JSON-RPC 2.0 风格** 消息，经 **AgentRouter** 寻址。  
> 与 [ARCHITECTURE_A2A_MCP.md](./260630_架构A2A_MCP.md) 配套使用。

---

## 1. 端点约定

| 组件 | 端点 | 说明 |
|------|------|------|
| AgentRouter | `POST /a2a/v1/router/dispatch` | 按 capability 路由到目标 Agent |
| AgentRegistry | `GET /a2a/v1/agents` | 列出 AgentCard |
| AgentRegistry | `GET /a2a/v1/agents/{name}/card` | 获取单张 AgentCard |
| Business Agent | `POST /a2a/v1/tasks/send` | 接收任务 |
| Business Agent | `GET /a2a/v1/tasks/{task_id}` | 查询 TaskState |
| Business Agent | `POST /a2a/v1/tasks/{task_id}/cancel` | 取消任务 |

---

## 2. AgentCard Schema

```yaml
# 最小必填
name: string              # 全局唯一，如 quality-rca-agent
description: string       # NL 描述，供 Planner/Router 匹配
url: string               # Agent 基址
version: string
capabilities: string[]    # 粗粒度，如 root_cause_analysis
enabled: boolean

# 推荐
skills:
  - id: string
    name: string
    description: string?
    input_schema: object  # JSON Schema
auth:
  type: bearer | mtls
  scopes: string[]
mcp_servers: string[]     # 依赖的 MCP Server id
hitl_required_below: float?  # 置信度阈值，如 0.7
factory_ids: string[]?    # 多厂隔离
```

## 3. 消息格式

### 3.1 发送任务 `tasks/send`

**Request**

```json
{
  "jsonrpc": "2.0",
  "id": "req-001",
  "method": "tasks/send",
  "params": {
    "task": {
      "id": "task-8f3a2b",
      "session_id": "sess-001",
      "trace_id": "tr-abc123",
      "capability": "root_cause_analysis",
      "message": {
        "role": "user",
        "parts": [
          { "type": "text", "text": "批次 B20260623001 涂布面密度偏低，请分析根因" }
        ]
      },
      "metadata": {
        "factory_id": "FD-01",
        "batch_id": "B20260623001",
        "defect_type": "coating_density_low",
        "prior_evidence": [],
        "playbook": "investigate"
      }
    }
  }
}
```

**Response**

```json
{
  "jsonrpc": "2.0",
  "id": "req-001",
  "result": {
    "task_id": "task-8f3a2b",
    "status": "RUNNING",
    "agent": "quality-rca-agent",
    "created_at": "2026-06-23T10:00:00Z"
  }
}
```

### 3.2 查询任务 `tasks/get`

```json
{
  "jsonrpc": "2.0",
  "id": "req-002",
  "method": "tasks/get",
  "params": { "task_id": "task-8f3a2b" }
}
```

```json
{
  "result": {
    "task_id": "task-8f3a2b",
    "status": "INPUT_REQUIRED",
    "agent": "quality-rca-agent",
    "artifacts": [
      {
        "type": "evidence_chain",
        "data": { "confidence": 0.62, "candidates": ["浆料粘度偏高", "涂布间隙偏小"] }
      }
    ],
    "hitl": {
      "level": "L1",
      "prompt": "置信度不足，请确认是否继续补查或采纳 Top-1",
      "thread_id": "hitl-thread-xyz"
    }
  }
}
```

### 3.3 HITL 恢复 `tasks/resume`

```json
{
  "method": "tasks/resume",
  "params": {
    "task_id": "task-8f3a2b",
    "thread_id": "hitl-thread-xyz",
    "feedback": {
      "action": "approve",
      "selected_root_cause": "浆料粘度偏高",
      "comment": "已与浆料房确认"
    }
  }
}
```

### 3.4 Agent 间委托 `tasks/delegate`（经 Router）

Planner 或业务 Agent 请求 Router 将子任务委派给其他 Agent：

```json
{
  "method": "tasks/delegate",
  "params": {
    "parent_task_id": "task-parent",
    "capability": "batch_trace",
    "message": {
      "role": "agent",
      "parts": [{ "type": "text", "text": "追溯批次 B20260623001 涂布段工艺参数" }]
    },
    "metadata": { "batch_id": "B20260623001", "scopes": ["coating", "slurry"] }
  }
}
```

Router 返回子 `task_id`，父任务通过 `parent_task_id` 关联。

## 4. TaskState 状态机

```
                    ┌─────────────┐
                    │  SUBMITTED  │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
              ┌────│   RUNNING   │────┐
              │    └──────┬──────┘    │
              │           │           │
       ┌──────▼───┐ ┌─────▼─────┐ ┌───▼────┐
       │ CANCELLED│ │INPUT_REQ  │ │ FAILED │
       └──────────┘ └─────┬─────┘ └────────┘
                          │ resume
                    ┌─────▼─────┐
                    │  RUNNING  │
                    └─────┬─────┘
                          │
                    ┌─────▼─────┐
                    │ COMPLETED │
                    └───────────┘
```

| 状态 | HTTP 轮询 | SSE 事件名 |
|------|-----------|------------|
| SUBMITTED | 202 | `task.submitted` |
| RUNNING | 200 | `task.running` |
| INPUT_REQUIRED | 200 | `task.input_required` |
| COMPLETED | 200 | `task.completed` |
| FAILED | 200 | `task.failed` |
| CANCELLED | 200 | `task.cancelled` |

## 5. 与 HTTP OpenAPI 双栈

| 场景 | 协议 | 说明 |
|------|------|------|
| 平台内同厂 | HTTP OpenAPI | P0：`POST /v1/analysis/quality` 等，兼容现有 RCA |
| 跨厂/跨团队 | A2A JSON-RPC | P1：AgentCard + Router |
| RCA 服务内 | LangGraph State | 不跨服务 |

现有 `Battery_Agent_DS` API 可作为 RCA Agent 的 **OpenAPI 适配层**；对外暴露 A2A 时增加薄适配器将 `tasks/send` 转为内部 `/v1/analysis/quality` 调用。

---

## 6. 安全

- 所有 A2A 请求携带 `Authorization: Bearer <JWT>` 与 `X-Trace-Id`
- Router 校验调用方是否有权委派目标 `capability`
- `tasks/delegate` 到 Safety Agent 需额外 `safety:approve` scope
- 响应 `artifacts` 按数据域过滤（行级权限）

## 7. Python AgentCard 注册示例

```python
from pydantic import BaseModel, Field

class AgentSkill(BaseModel):
    id: str
    name: str
    input_schema: dict = Field(default_factory=dict)

class AgentCard(BaseModel):
    name: str
    description: str
    url: str
    version: str = "1.0.0"
    capabilities: list[str]
    skills: list[AgentSkill] = []
    mcp_servers: list[str] = []
    enabled: bool = True

RCA_CARD = AgentCard(
    name="quality-rca-agent",
    description="锂电质量根因分析：MCP跨域取证+FMEA规则+HITL",
    url="http://rca-svc:8003/a2a/v1",
    capabilities=["root_cause_analysis", "evidence_chain", "report_8d_draft"],
    skills=[
        AgentSkill(
            id="analyze_quality",
            name="质量异常根因分析",
            input_schema={
                "type": "object",
                "properties": {
                    "batch_id": {"type": "string"},
                    "defect_type": {"type": "string"},
                    "user_query": {"type": "string"},
                },
                "required": ["batch_id"],
            },
        )
    ],
    mcp_servers=["mes", "scada", "erp", "lims", "knowledge"],
)
```
